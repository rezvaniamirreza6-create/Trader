# 🤖 ربات تلگرامی تحلیل تکنیکال (نسخه حرفه‌ای)

ربات تلگرامی کامل برای تحلیل تکنیکال بازار کریپتو و فارکس، با کیبورد
دائمی، پنل مدیریت تعاملی، و سیستم لایسنس کامل.

## ⚠️ توجه مهم قبل از استفاده

این ربات یک **ابزار آموزشی و تحلیلی** است، نه یک ماشین کسب سود تضمینی.
اندیکاتورهای تکنیکال صرفاً داده‌های گذشته را پردازش می‌کنند و **هیچ
تضمینی برای آینده‌ی قیمت ندارند**. اگر قصد دارید این ربات را به دیگران
ارائه دهید، صادق باشید درباره محدودیت‌هایش.

## ✨ امکانات

**برای کاربر:**
- کیبورد دائمی پایین صفحه (نیازی به تایپ دستور نیست)
- تحلیل ۶ اندیکاتوره: RSI، MACD، میانگین متحرک، باندهای بولینگر، استوکاستیک، ATR
- پوشش ۶ بازار: BTC/USD، ETH/USD، EUR/USD، GBP/USD، USD/JPY، XAU/USD (طلا)
- نمایش سطح ریسک/نوسان بازار در کنار تحلیل جهت‌دار
- سیستم لایسنس با تاریخ انقضای مشخص

**برای ادمین:**
- پنل مدیریت تعاملی با دکمه (دستور `/admin`)
- ساخت لایسنس با انتخاب مدت اعتبار از دکمه (۷ تا ۳۶۵ روز)
- لیست کامل لایسنس‌ها و کاربران
- آمار کامل سیستم (کاربران فعال، تعداد تحلیل‌ها، و غیره)
- مسدودسازی/رفع مسدودی کاربران (`/ban`, `/unban`)
- باطل کردن لایسنس (`/revoke`)
- ارسال پیام همگانی به تمام کاربران (Broadcast)

## 📁 ساختار پروژه

```
forex_bot/
├── bot.py              # فایل اصلی ربات (تمام منطق تلگرام)
├── license_manager.py  # مدیریت لایسنس، کاربران، آمار
├── analysis.py          # موتور تحلیل تکنیکال (۶ اندیکاتور)
├── requirements.txt     # پکیج‌های مورد نیاز
├── runtime.txt           # نسخه پایتون (برای سازگاری در Railway)

├── .gitignore            # جلوگیری از آپلود فایل‌های حساس
└── data/                  # (خودکار ساخته می‌شود) فایل‌های JSON
```

## 🚀 راه‌اندازی کامل (مرحله به مرحله)

### مرحله ۱: گرفتن توکن امن از تلگرام

1. به [@BotFather](https://t.me/BotFather) پیام بده
2. اگر ربات قبلی داشتی که توکنش جایی لو رفته: `/mybots` → ربات → API Token → Revoke
3. توکن جدید (یا توکن ربات تازه با `/newbot`) را کپی کن — **این را در کد قرار نمی‌دهیم**

### مرحله ۲: گرفتن کلید رایگان Twelve Data (برای داده قیمت)

این ربات از [Twelve Data](https://twelvedata.com) برای داده‌ی قیمت
استفاده می‌کند (نه Binance، چون سرورهای Railway توسط Binance مسدود
می‌شوند - خطای HTTP 451).

1. به [twelvedata.com/register](https://twelvedata.com/register) برو و ثبت‌نام کن (رایگان، بدون کارت بانکی)
2. ایمیلت را تایید کن
3. به [twelvedata.com/apikey](https://twelvedata.com/apikey) برو و کلیدت را کپی کن

> پلن رایگان معمولاً حدود ۸۰۰ درخواست در روز اجازه می‌دهد — برای
> استفاده شخصی و تست کاملاً کافی است.

### مرحله ۳: آپلود در گیت‌هاب

```bash
cd forex_bot
git init
git add .
git commit -m "Initial commit"
git remote add origin <لینک ریپوی شما>
git push -u origin main
```

فایل `.gitignore` تنظیم شده تا پوشه‌ی `data/` (اطلاعات کاربران و
لایسنس‌ها) هرگز آپلود نشود.

### مرحله ۴: دیپلوی روی Railway

1. در [railway.app](https://railway.app) یک پروژه جدید از روی ریپوی گیت‌هابت بساز
2. به تب **Variables** برو و این مقادیر را اضافه کن:

| نام متغیر | مقدار |
|---|---|
| `BOT_TOKEN` | توکن جدید از BotFather |
| `ADMIN_ID` | آیدی عددی تلگرام شما (نه یوزرنیم!) |
| `TWELVE_DATA_API_KEY` | کلیدی که از Twelve Data گرفتی |

3. Railway به‌صورت خودکار `requirements.txt` را نصب و `Procfile` را اجرا می‌کند

> برای پیدا کردن آیدی عددی تلگرامت، به [@userinfobot](https://t.me/userinfobot) پیام بده.

## 📖 راهنمای استفاده

### کاربر عادی
بعد از `/start`، یک کیبورد با ۴ دکمه پایین صفحه ظاهر می‌شود:
- **📊 تحلیل تکنیکال** — انتخاب بازار و دریافت تحلیل
- **🔑 فعال‌سازی لایسنس** — وارد کردن کد لایسنس (یا فقط کد را مستقیم تایپ کنید)
- **👤 وضعیت اشتراک من** — مشاهده تاریخ انقضا
- **ℹ️ راهنما** — توضیحات استفاده

### ادمین
ادمین (آیدی برابر با `ADMIN_ID`) یک دکمه‌ی پنجم هم می‌بیند:
- **🛠 پنل مدیریت** — باز کردن پنل با دکمه‌های:
  - ➕ ساخت لایسنس جدید
  - 📋 لیست لایسنس‌ها
  - 📈 آمار کامل سیستم
  - 👥 لیست کاربران
  - 📢 ارسال پیام همگانی

دستورات متنی ادمین (بدون نیاز به پنل):
```
/revoke XXXX-XXXX-XXXX-XXXX    باطل کردن یک کد لایسنس
/ban 123456789                  مسدود کردن کاربر
/unban 123456789                رفع مسدودیت کاربر
```

## 🔧 رفع مشکلات رایج

### خطای "Preparing metadata (pyproject.toml) ... error" روی Railway
دلیلش نسخه‌ی قدیمی pandas است که برای Python جدید wheel آماده ندارد.
این پروژه با `requirements.txt` (نسخه‌ی `>=`) و `runtime.txt`
(قفل پایتون 3.12) این مشکل را از قبل حل کرده است.

### خطای "Client Error 451" هنگام تحلیل
این یعنی هنوز جایی از کد به Binance متصل می‌شود. در نسخه‌ی فعلی همه‌چیز
از Twelve Data می‌آید، پس این خطا دیگر نباید رخ دهد. اگر دیدید، مطمئن
شوید آخرین نسخه‌ی `analysis.py` و `bot.py` را دارید.

### "شما دسترسی ادمین ندارید"
یعنی مقدار `ADMIN_ID` در Railway با آیدی عددی تلگرام شما یکی نیست.
از [@userinfobot](https://t.me/userinfobot) آیدی واقعی‌ات را بگیر و
در Railway → Variables → ADMIN_ID جایگزین کن (و سرویس را Redeploy کن).

### خطای 404 هنگام دریافت داده فارکس/کریپتو
فرمت سیمبل باید با اسلش باشد (`EUR/USD` نه `EURUSD`). در نسخه‌ی فعلی
این فرمت از قبل رعایت شده است.

## ⚖️ نکته حقوقی و اخلاقی

اگر این ربات را در اختیار دیگران (مشتری، دوست، کانال) قرار می‌دهید:
- هرگز ادعای "سود تضمینی" یا "دقت ۱۰۰٪" نکنید
- پیام افشای ریسک (disclaimer) که در هر تحلیل نمایش داده می‌شود را حذف نکنید
- در بسیاری از کشورها، تبلیغ کاذب ابزار مالی با ادعای سود قطعی، نقض
  قوانین حمایت از مصرف‌کننده محسوب می‌شود
