# 🤖 ربات تلگرامی تحلیل تکنیکال (نسخه حرفه‌ای)

ربات تلگرامی کامل برای تحلیل تکنیکال بازار کریپتو و فارکس، با کیبورد
دائمی، پنل مدیریت تعاملی، و سیستم لایسنس کامل.

## ⚠️ توجه مهم قبل از استفاده

این ربات یک **ابزار آموزشی و تحلیلی** است، نه یک ماشین کسب سود تضمینی.
اندیکاتورهای تکنیکال صرفاً داده‌های گذشته را پردازش می‌کنند و **هیچ
تضمینی برای آینده‌ی قیمت ندارند**. اگر قصد دارید این ربات را به دیگران
ارائه دهید، صادق باشید درباره محدودیت‌هایش.

## ✨ امکانات

**برای کاربر:**
- کیبورد دائمی پایین صفحه (نیازی به تایپ دستور نیست)
- موتور تحلیل ۷ بُعدی: RSI، MACD، میانگین متحرک، باندهای بولینگر، استوکاستیک، ATR، **ADX + Ichimoku**
- **تشخیص رژیم بازار** (روند‌دار / رنج / گذار) و **وزن‌دهی پویای اندیکاتورها** متناسب با رژیم — تفاوت اصلی تحلیل حرفه‌ای با رای‌گیری ساده آماتوری
- پوشش ۱۵ بازار:
  - **کریپتو**: بیت‌کوین، اتریوم، سولانا
  - **فارکس**: EUR/USD، GBP/USD، USD/JPY، USD/CHF، AUD/USD، USD/CAD
  - **کالا**: طلا، نقره، نفت خام
  - **شاخص‌های سهام آمریکا**: داوجونز (DJI)، اس‌اندپی ۵۰۰ (SPX)، نزدک (IXIC)
- **انتخاب تایم‌فریم تحلیل** — بعد از انتخاب بازار، کاربر تایم‌فریم را هم انتخاب می‌کند (۱۵ دقیقه، ۳۰ دقیقه، ۱ ساعت، ۴ ساعت، یا روزانه)
- **نمودار کندل‌استیک تصویری** همراه با هر تحلیل (شامل میانگین متحرک و باندهای بولینگر)
- نمایش سطح ریسک/نوسان بازار در کنار تحلیل جهت‌دار
- **پلن معاملاتی Entry/Stop Loss/Take Profit** برای ثبت دستی در MetaTrader (بر اساس ATR، نسبت ریسک‌به‌ریوارد ۱:۱.۵)
- **داشبورد عملکرد واقعی** — هر سیگنال ثبت و بعداً با قیمت واقعی بازار ارزیابی می‌شود (نه ادعای دقت ساختگی)
- **اخبار اقتصادی و هشدار ریسک خبری** — تشخیص اخبار پرریسک پیش رو (مثل تصمیم نرخ بهره فدرال، CPI، NFP) و ترکیب خودکار با تحلیل تکنیکال
- سیستم لایسنس با تاریخ انقضای مشخص

**برای ادمین:**
- پنل مدیریت تعاملی با دکمه (دستور `/admin`)
- ساخت لایسنس با انتخاب مدت اعتبار از دکمه (۷ تا ۳۶۵ روز)
- لیست کامل لایسنس‌ها و کاربران
- آمار کامل سیستم (کاربران فعال، تعداد تحلیل‌ها، و غیره)
- مسدودسازی/رفع مسدودی کاربران (`/ban`, `/unban`)
- باطل کردن لایسنس (`/revoke`)
- ارسال پیام همگانی به تمام کاربران (Broadcast)

## 🚀 راه‌اندازی کامل (مرحله به مرحله)

### مرحله ۱: گرفتن توکن امن از تلگرام

1. به [@BotFather](https://t.me/BotFather) پیام بده
2. اگر ربات قبلی داشتی که توکنش جایی لو رفته: `/mybots` → ربات → API Token → Revoke
3. توکن جدید (یا توکن ربات تازه با `/newbot`) را کپی کن — **این را در کد قرار نمی‌دهیم**

### مرحله ۲: گرفتن کلید رایگان Twelve Data (برای داده قیمت)

این ربات از [Twelve Data](https://twelvedata.com) برای داده‌ی قیمت
استفاده می‌کند (نه Binance، چون سرورهای Railway توسط Binance مسدود
می‌شوند - خطای HTTP 451).

1. به [twelvedata.com/register](https://twelvedata.com/register) برو و ثبت‌نام کن (رایگان، بدون کارت بانکی)
2. ایمیلت را تایید کن
3. به [twelvedata.com/apikey](https://twelvedata.com/apikey) برو و کلیدت را کپی کن

> پلن رایگان معمولاً حدود ۸۰۰ درخواست در روز اجازه می‌دهد — برای
> استفاده شخصی و تست کاملاً کافی است.

### مرحله ۳: گرفتن کلید رایگان Finnhub (برای اخبار اقتصادی)

این ربات از [Finnhub](https://finnhub.io) برای تقویم اقتصادی و هشدار
ریسک خبری استفاده می‌کند.

1. به [finnhub.io/register](https://finnhub.io/register) برو و ثبت‌نام کن (رایگان، بدون کارت بانکی)
2. بعد از ورود، کلیدت در صفحه‌ی Dashboard نمایش داده می‌شود — کپی‌اش کن

> پلن رایگان Finnhub اجازه‌ی ۶۰ درخواست در دقیقه می‌دهد — کاملاً کافی
> برای این ویژگی. اگر این کلید را تنظیم نکنید، ربات بدون مشکل کار
> می‌کند؛ فقط بخش اخبار و هشدار خبری غیرفعال می‌ماند.

### مرحله ۴: آپلود در گیت‌هاب

```bash
cd forex_bot
git init
git add .
git commit -m "Initial commit"
git remote add origin <لینک ریپوی شما>
git push -u origin main
```

فایل `.gitignore` تنظیم شده تا پوشه‌ی `data/` (اطلاعات کاربران و
لایسنس‌ها) هرگز آپلود نشود.

### مرحله ۵: دیپلوی روی Railway

1. در [railway.app](https://railway.app) یک پروژه جدید از روی ریپوی گیت‌هابت بساز
2. به تب **Variables** برو و این مقادیر را اضافه کن:

| نام متغیر | مقدار |
|---|---|
| `BOT_TOKEN` | توکن جدید از BotFather |
| `ADMIN_ID` | آیدی عددی تلگرام شما (نه یوزرنیم!) |
| `TWELVE_DATA_API_KEY` | کلیدی که از Twelve Data گرفتی |
| `FINNHUB_API_KEY` | کلیدی که از Finnhub گرفتی (اختیاری - بدون آن، بخش اخبار غیرفعال می‌ماند) |

3. Railway به‌صورت خودکار `requirements.txt` را نصب و `Procfile` را اجرا می‌کند

> برای پیدا کردن آیدی عددی تلگرامت، به [@userinfobot](https://t.me/userinfobot) پیام بده.

## 📖 راهنمای استفاده

### کاربر عادی
بعد از `/start`، یک کیبورد دائمی پایین صفحه ظاهر می‌شود:
- **📊 تحلیل تکنیکال** — انتخاب بازار، سپس انتخاب تایم‌فریم (۱۵ دقیقه تا روزانه)، سپس دریافت تحلیل
- **🔑 فعال‌سازی لایسنس** — وارد کردن کد لایسنس (یا فقط کد را مستقیم تایپ کنید)
- **👤 وضعیت اشتراک من** — مشاهده تاریخ انقضا
- **📈 داشبورد عملکرد** — آمار واقعی دقت سیگنال‌های گذشته
- **📰 اخبار اقتصادی** — رویدادهای پرریسک پیش رو و مقایسه‌ی واقعی/پیش‌بینی
- **ℹ️ راهنما** — توضیحات استفاده

### ادمین
ادمین (آیدی برابر با `ADMIN_ID`) یک دکمه‌ی اضافه هم می‌بیند:
- **🛠 پنل مدیریت** — باز کردن پنل با دکمه‌های:
  - ➕ ساخت لایسنس جدید
  - 📋 لیست لایسنس‌ها
  - 📈 آمار کامل سیستم
  - 👥 لیست کاربران
  - 📢 ارسال پیام همگانی

دستورات متنی ادمین (بدون نیاز به پنل):
```
/revoke XXXX-XXXX-XXXX-XXXX    باطل کردن یک کد لایسنس
/ban 123456789                  مسدود کردن کاربر
/unban 123456789                رفع مسدودیت کاربر
```

## 🔧 رفع مشکلات رایج

### چرا برچسب‌های نمودار تصویری انگلیسی است، نه فارسی؟
کتابخانه‌ی matplotlib بدون افزونه‌های اضافه (`arabic_reshaper` و
`python-bidi`) متن فارسی را به‌صورت معکوس و نادرست رسم می‌کند. برای
جلوگیری از این مشکل، عنوان و برچسب‌های نمودار عمداً به‌صورت لاتین/سیمبل
نوشته شده‌اند (مثل `BTC/USD | 1h`). توضیح کامل فارسی همیشه در پیام
متنی همراه نمودار ارسال می‌شود.

### تحلیل شاخص‌های سهام (داوجونز/اس‌اندپی/نزدک) داده‌ی کم یا خالی برمی‌گرداند
این شاخص‌ها برخلاف کریپتو، ۲۴/۷ معامله نمی‌شوند — فقط در ساعات کاری
بورس آمریکا (تقریباً ۱۳:۳۰ تا ۲۰:۰۰ به وقت UTC، در روزهای کاری) داده‌ی
زنده دارند. خارج از این بازه، ممکن است آخرین کندل‌های موجود تکراری
یا قدیمی باشند. ربات این محدودیت را در پیام تحلیل یادآوری می‌کند.

### خطای "Preparing metadata (pyproject.toml) ... error" روی Railway
دلیلش نسخه‌ی قدیمی pandas است که برای Python جدید wheel آماده ندارد.
این پروژه با `requirements.txt` (نسخه‌ی `>=`) و `runtime.txt`
(قفل پایتون 3.12) این مشکل را از قبل حل کرده است.

### خطای "Client Error 451" هنگام تحلیل
این یعنی هنوز جایی از کد به Binance متصل می‌شود. در نسخه‌ی فعلی همه‌چیز
از Twelve Data می‌آید، پس این خطا دیگر نباید رخ دهد. اگر دیدید، مطمئن
شوید آخرین نسخه‌ی `analysis.py` و `bot.py` را دارید.

### "شما دسترسی ادمین ندارید"
یعنی مقدار `ADMIN_ID` در Railway با آیدی عددی تلگرام شما یکی نیست.
از [@userinfobot](https://t.me/userinfobot) آیدی واقعی‌ات را بگیر و
در Railway → Variables → ADMIN_ID جایگزین کن (و سرویس را Redeploy کن).

### خطای 404 هنگام دریافت داده فارکس/کریپتو
فرمت سیمبل باید با اسلش باشد (`EUR/USD` نه `EURUSD`). در نسخه‌ی فعلی
این فرمت از قبل رعایت شده است.

### خطای "JobQueue is not available" یا داشبورد عملکرد همیشه خالی است
از نسخه‌ی فعلی `requirements.txt` استفاده کنید — باید دقیقاً این خط را
داشته باشد: `python-telegram-bot[job-queue]==21.0.1` (نه بدون `[job-queue]`).
این extra برای اجرای ارزیابی خودکار سیگنال‌ها هر ۱۵ دقیقه لازم است.
اگر بعد از این هم مشکل ادامه داشت، روی Railway سرویس را Redeploy کنید
تا cache نصب پکیج‌ها پاک شود.

## ⚖️ نکته حقوقی و اخلاقی

اگر این ربات را در اختیار دیگران (مشتری، دوست، کانال) قرار می‌دهید:

- در بسیاری از کشورها، تبلیغ کاذب ابزار مالی با ادعای سود قطعی، نقض

## 📐 نحوه استفاده از پلن معاملاتی (Entry/SL/TP) در MetaTrader

ربات برای هر سیگنال جهت‌دار (غیر خنثی)، یک پلن معاملاتی شامل سه عدد
محاسبه می‌کند:

- **Entry**: قیمت فعلی لحظه‌ی تحلیل
- **Stop Loss**: فاصله‌اش ۱.۵ برابر ATR (میانگین نوسان واقعی اخیر بازار)
- **Take Profit**: فاصله‌اش ۱.۵ برابر فاصله‌ی Stop Loss (نسبت ریسک‌به‌ریوارد ۱:۱.۵)

این اعداد را **خودتان دستی** در MetaTrader وارد می‌کنید:
1. نماد مربوطه را در Market Watch پیدا کنید (مثلاً `BTCUSD` برای BTC/USD)
2. New Order را باز کنید
3. نوع سفارش را مطابق جهت پیشنهادی (Buy/Sell) انتخاب کنید
4. مقادیر Stop Loss و Take Profit را از پیام ربات کپی کنید
5. حجم معامله (لات) را خودتان بر اساس مدیریت ریسک شخصی‌تان تعیین کنید

**⚠️ مهم:** این اعداد فاصله‌ی خروج منطقی از معامله را مشخص می‌کنند،
نه احتمال درست بودن جهت معامله. مدیریت حجم و ریسک معامله بر عهده‌ی
کامل خود شماست.

## 🧠 موتور تحلیل حرفه‌ای: رژیم‌بندی بازار و وزن‌دهی پویا

نسخه فعلی موتور تحلیل، به‌جای رای‌گیری ساده و هم‌وزن بین اندیکاتورها
(که روش تحلیلگرهای آماتور است)، دو مرحله انجام می‌دهد:

**مرحله ۱ — تشخیص رژیم بازار با ADX:**
قبل از هر چیز، با شاخص ADX بررسی می‌شود که آیا بازار در حال حاضر یک
روند مشخص دارد (Trending) یا در حال نوسان بدون جهت (Ranging) است.

**مرحله ۲ — وزن‌دهی پویا بر اساس رژیم:**
- در بازار **روند‌دار**: اندیکاتورهای روندی (MACD، میانگین متحرک،
  Ichimoku) وزن بیشتری در تصمیم نهایی می‌گیرند.
- در بازار **رنج**: اسیلاتورهای اشباع (RSI، استوکاستیک، بولینگر)
  وزن بیشتری می‌گیرند.

این دقیقاً تفاوت اصلی یک تحلیل حرفه‌ای با تحلیل آماتوری است: استفاده
از یک اندیکاتور روندی در بازار بی‌روند (یا برعکس) می‌تواند گمراه‌کننده
باشد، و تحلیلگر حرفه‌ای همیشه اول می‌پرسد «این اصلاً چه نوع بازاری است؟»

### 📊 صداقت آماری: نتیجه واقعی بک‌تست

این موتور پیشرفته روی بیش از ۹۸۰۰ سیگنال شبیه‌سازی‌شده آزمایش شد:

| نسخه موتور | دقت کلی |
|---|---|
| نسخه ۱ (رای‌گیری ساده، ۵ اندیکاتور هم‌وزن) | ۴۸.۲٪ |
| نسخه ۲ (رژیم‌بندی ADX + وزن‌دهی پویا + Ichimoku) | ۴۹.۵٪ |

تفاوت تنها ۱.۳ درصد است — درون محدوده‌ی نوسان آماری، نه یک جهش
معنادار. این نتیجه با نظریه‌ی «بازار کارا» (Efficient Market
Hypothesis) هم‌خوانی دارد: اندیکاتورهای تکنیکال کلاسیک به‌تنهایی
مزیت پیش‌بینی پایداری بالاتر از شانس تصادفی ایجاد نمی‌کنند، حتی با
مهندسی پیچیده‌تر. ارزش واقعی نسخه‌ی جدید در **کیفیت و عمق تحلیل
ارائه‌شده** است (تشخیص رژیم بازار، وضوح در توضیح چرایی هر سیگنال)،
نه در افزایش قطعیت پیش‌بینی که از نظر آماری وجود ندارد.

## 📈 داشبورد عملکرد واقعی (Performance Dashboard)

به‌جای ادعای یک عدد دقت ثابت و فرضی، ربات اکنون **هر سیگنالی که صادر
می‌کند را ثبت می‌کند** و بعداً با گرفتن قیمت واقعی بازار، نتیجه‌ی
واقعی آن را ارزیابی می‌نماید:

- **ارزیابی کوتاه‌مدت**: ۵ کندل بعد از سیگنال (با تایم‌فریم ۱ ساعته، یعنی ۵ ساعت بعد)
- **ارزیابی بلندمدت**: ۲۴ ساعت بعد از سیگنال

این ارزیابی به‌صورت خودکار هر ۱۵ دقیقه توسط یک Job پشت‌صحنه اجرا
می‌شود (`evaluate_pending_signals_job` در `bot.py`). کاربر با دکمه‌ی
**«📈 داشبورد عملکرد»** یا دستور `/dashboard` می‌تواند دقت واقعی و
به‌روز ربات را — برآمده از داده‌ی واقعی، نه ادعا — ببیند.

**چرا این مهم است؟** هر عدد دقتی که از قبل در کد هاردکد شده باشد،
لزوماً نادرست یا گمراه‌کننده است (چون بازار واقعی، شرایط متغیر، و
نوسانات هرلحظه عوض می‌شوند). داشبورد عملکرد این مشکل را با اندازه‌گیری
**مستقیم و زنده** حل می‌کند. انتظار داشته باشید این عدد در بلندمدت
نزدیک به ۵۰٪ بماند — این طبیعی و مطابق با واقعیت آماری بازارهای
مالی کارا است، نه یک باگ.

## 📰 اخبار اقتصادی و هشدار ریسک خبری

ربات اکنون تقویم اقتصادی را پایش می‌کند (با [Finnhub](https://finnhub.io))
و دو کار مشخص انجام می‌دهد:

1. **قبل از انتشار خبر مهم** (مثل تصمیم نرخ بهره فدرال، CPI، NFP،
   تصمیمات بانک مرکزی): هشدار می‌دهد که در فلان ساعت، نوسان شدید
   احتمال دارد — این هشدار **زمان** ریسک را نشان می‌دهد، نه **جهت** آن.
2. **بعد از انتشار خبر**: عدد واقعی (Actual) را با پیش‌بینی بازار
   (Forecast) مقایسه می‌کند و می‌گوید بهتر یا بدتر از انتظار بود —
   این یک فکت عینی است (محاسبه‌ی ساده‌ی دو عدد)، نه پیش‌بینی.

این هشدار به‌صورت خودکار با تحلیل تکنیکال هم ترکیب می‌شود: وقتی روی
دکمه‌ی تحلیل یک بازار کلیک می‌کنید، اگر خبر پرریسکی در ۶ ساعت آینده
در راه باشد، بالای پیام تحلیل نمایش داده می‌شود.

دسترسی جداگانه به اخبار: دکمه‌ی **«📰 اخبار اقتصادی»** یا دستور `/news`.

**⚠️ محدودیت صادقانه:** این ماژول **نتیجه‌ی** خبر روی قیمت را پیش‌بینی
نمی‌کند و نمی‌تواند. واکنش بازار به یک خبر به عوامل غیرقابل‌پیش‌بینی
بسیاری بستگی دارد (آیا خبر غافلگیرکننده بود؟ بازار از قبل قیمت را
لحاظ کرده بود؟ واکنش روانی لحظه‌ای معامله‌گران چطور است؟). آنچه این
ماژول واقعاً انجام می‌دهد، اطلاع‌رسانی **زمان ریسک** و مقایسه‌ی
**فکت‌محور** عدد واقعی با پیش‌بینی است — نه فال‌گویی.
